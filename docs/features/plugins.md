# Feature: Plugins (Experimental)

## Overview

Plugins extend Forgedocs with custom pages, sidebar items, and hooks into discovery and build phases. They are ESM modules that receive a plugin API object.

**Status:** Experimental. The API may change between minor versions. No built-in plugins ship with Forgedocs yet. See [ADR-005](../adr/005-plugin-system-design.md) for design rationale.

## Plugin Format

A plugin is an ESM module that exports a default function:

```js
// my-plugin.mjs
export default function myPlugin(api, options) {
  api.addPages([{
    path: 'changelog.md',
    content: '# Changelog\n\nGenerated by plugin.',
  }])
}
```

The function receives:
- `api` — the plugin context object (see API Reference below)
- `options` — user-provided options from `docsite.config.mjs`

## Configuration

Register plugins in `docsite.config.mjs`:

```js
export default {
  plugins: [
    'forgedocs-plugin-openapi',                        // npm package
    ['forgedocs-plugin-mermaid', { theme: 'dark' }],   // with options
    './my-local-plugin.mjs',                           // local file
  ]
}
```

**Resolution order:**
- Paths starting with `./` or `/` — resolved relative to CWD
- Other strings — resolved from `node_modules/` in CWD, then as a bare ESM import

## API Reference

Source: `lib/plugins.mjs`

### `api.addPages(pages)`

Add virtual markdown pages to the doc site. Each page needs `path` (relative to content root) and `content` (markdown string).

```js
api.addPages([
  { path: 'generated/api-docs.md', content: '# API Docs\n...' },
])
```

### `api.addSidebarItems(service, items)`

Add sidebar entries to a specific service's navigation.

```js
api.addSidebarItems('my-api', [
  { text: 'API Reference', link: '/generated/api-docs' },
])
```

### `api.addGlobalSidebarItems(items)`

Add sidebar entries to all services' navigation.

```js
api.addGlobalSidebarItems([
  { text: 'Changelog', link: '/changelog' },
])
```

### `api.onDiscover(fn)`

Hook called during service discovery. Receives the `services` array — can inspect or mutate it.

```js
api.onDiscover((services) => {
  console.log(`Discovered ${services.length} services`)
})
```

### `api.onBuild(fn)`

Hook called during site build. Receives `buildInfo` with build context.

```js
api.onBuild((buildInfo) => {
  console.log('Build started')
})
```

## Exported Functions

| Function | Purpose |
|----------|---------|
| `loadPlugins(specs, cwd)` | Load and initialize all plugins, returns unified context |
| `writePluginPages(ctx, contentDir)` | Write plugin-generated pages to disk |
| `getPluginSidebarItems(ctx, service)` | Get extra sidebar items for a service |
| `runDiscoverHooks(ctx, services)` | Run all discovery hooks |
| `runBuildHooks(ctx, buildInfo)` | Run all build hooks |

## Error Handling

- If a plugin fails to load, a warning is printed but other plugins continue loading.
- If a plugin does not export a default function, a warning is printed.
- Hook errors are caught individually — one failing hook does not block others.
